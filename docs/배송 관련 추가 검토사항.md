## 확정된 배송 구조 (2026-03-20)

### 1. 배송 방법 — 하이브리드 3모델 병행

| 모델 | 방식 | 대상 지역 | 특징 |
|------|------|----------|------|
| Model A: 꽃차 직배송 (Direct) | 드라이버 앱 연동, 판매자 직접 운행 | 수도권 (서울/경기) | 전 품목, 전용 냉장/항온 차량 |
| Model B: 거점 픽업 (Hub) | 협력 업체(꽃집, 과일가게 등)에 대량 배송 후 소비자 픽업 | 수도권 (서울/경기) | 물류비 절감, 소비자 수령 유연성 |
| Model C: 표준 택배 (Parcel) | 외부 택배사 연동 | 전국 | 포장 표준화 상품 한정, 기상 조건 가변 운영 |

> 결정: **하이브리드 3모델 확정** ✅

---

### 2. 배송 범위 / 가능 지역 — isMetropolitan 기반 자동 분기

- **수도권 (서울/경기)**: Model A (꽃차 직배송) + Model B (거점 픽업) + Model C (택배) 모두 가능
- **수도권 외 전국**: Model C (택배)만 가능
- **시스템 구현**: 소비자 앱 주소 입력 시 주소 API 연동으로 `isMetropolitan` 즉시 판별 → 사용 가능한 배송 수단 필터링 UI 표시

> 결정: **isMetropolitan 속성으로 배송 수단 자동 분기** ✅

---

### 3. 배송비 정책 — 원칙 확정, 단가 미정

**확정된 원칙:**

| 배송 유형 | 정책 |
|----------|------|
| 꽃차 직배송 / 택배 | 표준 배송비 + 일정 금액 이상 조건부 무료 검토 |
| 거점 픽업 (Hub) | 직배송 대비 저렴하게 책정, 소비자 방문 유도 |
| 공동구매 | 일반 배송비 대비 할인 배송비 적용. 이벤트 시 무료 설정 가능 (판매 촉진 도구) |
| 일반 판매 | 배송 수단 및 장바구니 합산 금액에 따른 자동 계산 |

**MVP 임시 기본값 (운영 중 판매자 앱에서 수정 가능):**

| 배송 유형 | 임시 배송비 | 무료 배송 기준 |
|----------|-----------|--------------|
| 꽃차 직배송 | 3,000원 | 50,000원 이상 무료 |
| 거점 픽업 | 1,000원 | 30,000원 이상 무료 |
| 택배 | 4,000원 | 50,000원 이상 무료 |
| 공동구매 | 할인 배송비 (판매자 설정) | 이벤트 시 무료 설정 가능 |

**공동구매 배송 수단 (소비자 선택 불가, 판매자가 상품 등록 시 지정):**
- 서울/경기: 꽃차 직배송 또는 택배 중 판매자가 지정
- 기타 전국: 택배 고정
- 배송비: 일반 배송비 대비 할인 배송비 적용 (판매자가 상품 등록 시 설정)
  → 이벤트/프로모션 시 무료 배송 설정 가능 (소비자 참여 유도 및 판매 촉진 도구)

**품목/사이즈별 배송비 차등 (판매자 앱에서 상품 등록 시 설정):**

| 사이즈 | 추가 배송비 | 예시 품목 |
|--------|-----------|---------|
| 소형 | 기본 배송비 | 꽃다발, 소형 화분 |
| 중형 | 기본 배송비 + 1,000원 | 꽃바구니, 중형 화분 |
| 대형 | 기본 배송비 + 3,000원~ | 화환, 대형 식물 |

**배송비 수정 기능 요구사항:**
- 판매자 앱: 상품 등록/수정 시 품목별 사이즈 지정 및 추가 배송비 설정
- 판매자 앱: 배송 유형별 기본 배송비 및 무료 배송 임계치 전역 수정

> 결정: **임시 단가 확정 + 품목/사이즈별 차등 구조 확정** ✅

### 4. 실시간 배송 추적 — 메시지 피드 방식

- **지도 실시간 추적 아님** → 드라이버 앱 상태 변경 이벤트 발생 시 타임라인 피드 업데이트
- 밀크런 경로 최적화 엔진 탑재 (드라이버 앱 측 기능)
- **거점 픽업 수령 인증**: QR 스캔 대신 **픽업 코드(주문번호 6자리)** 방식으로 MVP 운영

```
소비자: 앱에서 "픽업 코드 123456" 확인
거점 업체: 코드 대조 후 상품 전달 (별도 앱 불필요)
드라이버: 배달 완료 후 드라이버 앱에서 상태 업데이트
```

> 결정: **타임라인 피드 + 픽업 코드 방식** ✅

---

### 5. 배송기사 배정 방식 — MVP: 판매자 직접 수락

- 자동 배정 알고리즘 없음
- 드라이버 앱에서 주문 목록 확인 후 판매자가 직접 '수락(Accept)' 하여 배송 수행
- 운행 데이터 축적 → 향후 외부 드라이버 모집 시 정산/배정 알고리즘 기초 자료로 활용

> 결정: **MVP: 판매자 직접 수락 방식** ✅
