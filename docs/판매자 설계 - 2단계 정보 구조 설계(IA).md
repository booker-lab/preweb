## 1. 사이트맵 (Sitemap)

### Main Navigation (하단 탭 — 모바일 / 사이드바 — PC)

```
├── 주문 (Orders)             ← 기본 진입 화면
│     신규 주문 실시간 수신, 상태별 탭 필터
│
├── 상품 (Products)
│     상품 목록, 등록, 수정, 활성/비활성
│
├── 정산 (Settlements)
│     일별 요약, 기간별 조회, 주문별 상세
│
├── 거점 (Hubs)
│     거점 목록, 등록/수정, 픽업 코드 확인
│
└── 설정 (Settings)
      배송비 설정, 기상 제한 토글, Daily Cap 관리
      프로필 수정
```

### Admin Navigation (role: 'admin' 전용 — 일반 판매자 접근 불가)

```
└── /admin
      /admin/stores        판매자 목록 · 초대 토큰 발급 · 승인 · 수수료율 설정
      /admin/users         소비자 계정 조회 · 정지/복구
      /admin/orders        전체 주문 조회 · 환불 강제 처리
      /admin/settlements   판매자별 정산 처리 (이체 완료 처리)
      /admin/invite        초대 토큰 발급 (판매자 등록)
```

> **구조 로드맵**: 지금은 `apps/seller /admin`으로 통합 운영(B안). 운영 규모 확장 시 `apps/admin` 별도 앱으로 분리(A안). 분리 시 NestJS API 변경 없이 페이지 파일 이동만으로 완료.

### Sub Screens (상세 및 기능 화면)

```
├── 로그인 (/login)                    ← 미로그인 시 전체 리다이렉트
│     이메일 로그인 / 카카오 OAuth
│
├── 온보딩 (/onboarding)               ← 최초 로그인 1회만
│     사업자 프로필 미완성 시 전체 화면 차단(Guard)
│
├── 주문 상세 (/orders/[id])
│     상품명·금액·배송정보, 상태 전환 버튼, 취소 모달
│
├── 상품 등록·수정 (/products/new, /products/[id]/edit)
│     일반 판매 / 공동구매 분기 폼
│
├── Daily Cap 달력 (/settings/daily-caps)
│     날짜별 슬롯 시각화 및 수정
│
└── 픽업 코드 확인 (/hubs/[id]/pickup)
      6자리 수동 입력 또는 바코드 스캔
```

## 2. 사용자 이동 경로 (User Flow)

### A. 신규 판매자 온보딩

```
앱 최초 접근
    ↓
로그인 화면 (/login)
  - 이메일 로그인 / 카카오 OAuth
  - role: 'seller' 명시 (관리자가 사전에 stores 문서 + storeId 공유)
    ↓
[최초 로그인] → 온보딩 화면 (/onboarding)
  - 상호명, 사업자등록번호, 대표자명, 연락처, 소재지
  - 로고 이미지 업로드 (Firebase Storage → logoUrl 저장)
  - "저장 후 시작하기" → PATCH /stores/:storeId 호출
    ↓
[프로필 완성] → 주문 대시보드 (/orders) 진입
[프로필 미완성 재접속] → 온보딩 화면으로 강제 리다이렉트 (Guard)
```

### B. 신규 주문 처리 (일반 판매)

```
[Firestore 실시간 리스너] 신규 주문 수신
    ↓
푸시 알림 + 주문 탭 배지(Badge) 갱신
    ↓
주문 탭 → "처리 필요" 탭 (ACCEPTED 상태)
    ↓
주문 카드 클릭 → 주문 상세 화면 (/orders/[id])
  - 상품명, 소비자명, 결제금액, 배송수단, 주문시각 확인
    ↓
"준비 시작" 버튼 클릭
  - (선택) 드라이버 수거 예정 시간 입력 → preparedAt 저장
  - PATCH /stores/:storeId/orders/:orderId/status { status: 'PREPARING' }
    ↓
→ 주문 목록 "준비 중" 탭으로 이동
```

### C. 신규 주문 처리 (공동구매)

```
[Firestore 실시간 리스너] 공동구매 참여 수신 (RECRUITING 상태)
    ↓
목표 인원 달성 시 자동 → CONFIRMED 상태
  - 알림 발송: "OO 공동구매 목표 달성! 준비를 시작하세요."
    ↓
"처리 필요" 탭 (CONFIRMED 상태)
    ↓
"준비 시작" 버튼 → PREPARING 전환 (일반 판매와 동일)
```

### D. 강제 취소 처리

```
주문 상세 화면 → "강제 취소" 버튼
    ↓
취소 사유 입력 모달
  - 사유 텍스트 입력 필수
    ↓
확인 → PATCH .../status { status: 'CANCELLED', reason }
    ↓
소비자에게 카카오 알림톡 자동 발송 (사유 포함)
    ↓
→ 주문 목록 "취소" 탭으로 이동
```

### E. 상품 등록

```
상품 탭 → "+ 상품 등록" 버튼 → /products/new
    ↓
판매 방식 선택 (첫 번째 선택)
  ┌── 일반 판매
  │     상품명, 가격, 카테고리, 색상(멀티), 배송사이즈
  │     이미지 업로드 (Firebase Storage)
  │     "등록" → POST /stores/:storeId/products
  │
  └── 공동구매
        위 공통 필드 +
        최소·최대 인원, 모집 마감일, 배송 예정일
        배송 수단, 할인 배송비
        "등록" → POST /stores/:storeId/products (saleType: 'group')
    ↓
상품 목록으로 복귀 (등록된 상품 최상단 노출)
```

### F. 거점 픽업 코드 확인

```
거점 탭 → 거점 선택
    ↓
"픽업 대기 주문 목록" (status: HUB_ARRIVED)
    ↓
소비자와 코드 대조 방식 선택
  ┌── 바코드 스캔 (카메라 권한 요청)
  └── 6자리 수동 입력
    ↓
코드 일치 → PATCH .../pickup-confirm
    ↓
주문 상태 → PICKED_UP (픽업 완료)
소비자 픽업 완료 알림톡 발송 (없음 — 소비자가 직접 수령)
```

### G. 정산 확인

```
정산 탭 → 일별 요약 화면 (기본: 오늘)
  - 완료 건수, 총 매출, 정산 예정액 확인
    ↓
달력 필터 선택 → 기간별 조회
  - 총 매출 / 수수료 / 정산금 합계
    ↓
개별 주문 행 클릭 → 주문별 상세
  - 상품가 + 배송비 - 수수료 = 정산금 내역 확인
    ↓
(Should Have) "CSV 다운로드" 버튼
```

## 3. 상세 화면별 주요 구성 요소

### 1. 로그인 (/login)

```
로고 + 앱명 "Green Hub 판매자"

로그인 옵션
  - [카카오로 시작하기] (Kakao OAuth)
  - ─────── 또는 ───────
  - 이메일 입력 + 비밀번호 입력
  - [로그인] 버튼
  - 하단: 계정이 없으신가요? (관리자 문의 안내 — Phase 2 전까지 자체 가입 불가)
```

---

### 2. 온보딩 (/onboarding)

```
진행 단계 표시 (Step 1/1)

필수 입력 필드 (미입력 시 저장 불가)
  - 상호명 *
  - 대표자명 *
  - 연락처 * (010-0000-0000 포맷)
  - 소재지 * (주소 검색 API)

선택 입력 필드 (없어도 active 전환 가능, 설정에서 추후 입력)
  - 사업자등록번호 (000-00-00000 포맷 자동 적용)
  - 로고 이미지 업로드 (미업로드 시 기본 이미지)

[저장 후 시작하기] 버튼
  → 유효성 검사 통과 시 PATCH /stores/:storeId 호출 → /orders 이동
```

### 3. 주문 대시보드 (/orders)

```
헤더
  - 좌: "주문 관리"
  - 우: 알림 아이콘 + 실시간 접속 상태 점(●)

상태별 탭 필터 (5개)
  처리 필요 | 준비 중 | 배송 중 | 완료 | 취소

각 탭의 배지(Badge): 해당 상태 주문 건수 실시간 표시

주문 카드 (Firestore 실시간 리스너)
  ┌─────────────────────────────────┐
  │ [상태 뱃지]  상품명             │
  │ 소비자명 · 결제금액 · 배송수단  │
  │ 주문시각 (예: 13분 전)          │
  └─────────────────────────────────┘

빈 상태: "현재 해당 주문이 없습니다" 안내 문구
```

**탭별 포함 상태**

| 탭 | 포함 상태 코드 |
|----|---------------|
| 처리 필요 | ACCEPTED, CONFIRMED |
| 준비 중 | PREPARING |
| 배송 중 | DELIVERING, HUB_ARRIVED |
| 완료 | DELIVERED, PICKED_UP, REVIEWED |
| 취소 | CANCELLED |

---

### 4. 주문 상세 (/orders/[id])

```
헤더: "주문 #ID 뒷 6자리" + 뒤로가기

주문 정보 섹션
  - 상품명 · 수량 · 상품 금액
  - 배송비
  - 총 결제금액 (굵게)
  - 주문 시각

배송 정보 섹션
  - 소비자명 · 연락처
  - 배송 수단 (직배송 / 거점 픽업 / 택배)
  - 배송 희망일 또는 배송 예정일
  - [거점 픽업] 거점명 + 픽업 코드

상태 전환 버튼 영역 (현재 상태에 따라 조건부 노출)

  [처리 필요 상태 — ACCEPTED / CONFIRMED]
    "준비 시작" (Primary) → PREPARING 전환
      └── 선택: 드라이버 수거 예정 시간 입력 (DateTimePicker)
    "강제 취소" (Destructive) → 취소 모달

  [준비 중 상태 — PREPARING]
    [드라이버가 배송 중 처리 → 자동 전환 / 드라이버 앱 담당]
    "강제 취소" (Destructive) → 취소 모달

  [배송 중 / 거점 도착 상태]
    상태 안내 문구만 표시 (판매자 전환 불가)

  [완료 / 취소 상태]
    읽기 전용 표시

취소 모달
  - 취소 사유 입력 (텍스트, 최소 5자)
  - [취소 확정] / [닫기]
```

### 5. 상품 관리 (/products)

```
헤더
  - 좌: "상품 관리"
  - 우: [+ 등록] 버튼

필터/정렬 바
  - 전체 | 판매 중 | 비활성

상품 카드 목록
  ┌─────────────────────────────────┐
  │ [상품 이미지]  상품명           │
  │ 가격 · 카테고리 · 판매 방식    │
  │ [활성/비활성 토글]  [수정] [삭제]│
  └─────────────────────────────────┘

삭제: 확인 모달 → DELETE /stores/:storeId/products/:id
활성/비활성 토글: PATCH /stores/:storeId/products/:id/active
```

---

### 6. 상품 등록·수정 (/products/new, /products/[id]/edit)

```
판매 방식 선택 탭 (등록 시 첫 선택, 수정 시 변경 불가)
  [일반 판매]  [공동구매]

── 공통 필드 ──
  - 상품명 *
  - 가격 (원) *
  - 카테고리 * (절화 / 난 / 관엽)
  - 색상 * (멀티 선택 — 최대 5개)
      레드 핑크 화이트 옐로우 오렌지 퍼플 블루 그린 무늬 브라운 베이지 블랙 그레이
  - 배송 사이즈 * (small / medium / large)
  - 이미지 업로드 (최대 5장, Firebase Storage)

── 공동구매 전용 필드 (saleType: group) ──
  - 최소 참여 인원 *
  - 최대 참여 인원 *
  - 모집 마감일 *
  - 배송 예정일 *
  - 배송 수단 * (꽃차 직배송 / 택배)
  - 할인 배송비 (원, 0 입력 시 무료)

[저장] 버튼
```

### 7. Daily Cap 관리 (/settings/daily-caps)

```
헤더
  - "배송 슬롯 관리"
  - 월/년 이동 버튼 (< 2026년 3월 >)

달력 (월별)
  각 날짜 셀:
    ┌────────────┐
    │     25     │
    │  30 / 30   │  ← 잔여슬롯 / 총슬롯
    │  [SOLD OUT]│  (잔여 0 시)
    └────────────┘
  - 오늘 날짜 강조 표시
  - 과거 날짜: 읽기 전용 (실적 조회용)

날짜 선택 → 슬롯 수정 Sheet (하단 슬라이드)
  - 현재 값 표시
  - +/- 버튼 또는 직접 입력
  - [저장] → PATCH /stores/:storeId/daily-caps/:date

월별 데이터: GET /stores/:storeId/daily-caps?from=&to=
```

---

### 8. 배송 설정 (/settings/delivery)

```
헤더: "배송 설정"

── 배송비 설정 ──
  배송 수단별 섹션:

  [꽃차 직배송]
    기본 배송비: ____ 원
    무료 기준 금액: ____ 원 이상 시 무료

  [거점 픽업]
    기본 배송비: ____ 원
    무료 기준 금액: ____ 원 이상 시 무료

  [택배]
    기본 배송비: ____ 원
    무료 기준 금액: ____ 원 이상 시 무료

  [저장] → PATCH /stores/:storeId/delivery-config

── 기상 제한 컨트롤러 ──
  ┌────────────────────────────────────────┐
  │  기상 제한 모드                 [ON/OFF]│
  │  활성화 시 소비자 결제 화면의          │
  │  택배 옵션이 자동으로 비활성됩니다.    │
  │  직배송·거점 픽업은 영향 없습니다.     │
  └────────────────────────────────────────┘
  토글 → PATCH /stores/:storeId/delivery-config { weatherRestrictionActive }
```

---

### 9. 정산/매출 관리 (/settlements)

```
헤더
  - 좌: "정산 관리"
  - 우: (Should Have) [CSV 다운로드] 버튼

── 일별 요약 카드 (기본: 오늘) ──
  ┌────────────────────────────────────┐
  │  2026년 3월 28일 (금)              │
  │  완료 건수:  12건                  │
  │  총 매출:    ₩ 1,240,000           │
  │  정산 예정:  ₩ 1,240,000  (수수료 0%)│
  └────────────────────────────────────┘

── 기간별 조회 ──
  달력 필터 (시작일 ~ 종료일 선택)
  [조회] 버튼
  → 기간 합계 카드:
      총 매출 / 수수료 / 정산금

── 주문별 상세 목록 ──
  각 행:
    상품명 · 결제시각 · 상품가 + 배송비 - 수수료 = 정산금

API
  GET /stores/:storeId/settlements?from=&to=
  GET /stores/:storeId/settlements/summary?date=
```

## 4. 주문 상태 관리 매핑

### 판매자 앱 기준 상태 전환 권한

| 상태 코드 | 판매자 노출 명칭 | 전환 가능 액션 | 전환 주체 |
|-----------|----------------|---------------|----------|
| PENDING | (미노출) | - | 시스템 (내부) |
| RECRUITING | 모집 중 | - | 시스템 (공동구매) |
| ACCEPTED | 결제 완료 | 준비 시작 → PREPARING, 강제 취소 | **판매자** |
| CONFIRMED | 주문 확정 | 준비 시작 → PREPARING, 강제 취소 | **판매자** |
| PREPARING | 준비 중 | 강제 취소 | **판매자** (배송 전환은 드라이버) |
| DELIVERING | 배송 중 | - | 드라이버 앱 |
| HUB_ARRIVED | 거점 도착 | - | 드라이버 앱 |
| PICKED_UP | 픽업 완료 | - | **판매자** (픽업 코드 확인) |
| DELIVERED | 배송 완료 | - | 드라이버 앱 |
| CANCELLED | 취소 | - | 판매자 / 소비자 / 시스템 |
| REVIEWED | 구매 확정 | - | 소비자 앱 |

### 정산 레코드 자동 생성 시점

```
REVIEWED, DELIVERED, PICKED_UP 도달 시
  → settlements 컬렉션에 레코드 자동 생성 (status: 'confirmed')
  → commissionRate 스냅샷 저장 (나중에 요율 변경돼도 정산금 불변)

CANCELLED 전환 시
  → settlements.status = 'cancelled' 즉시 반영 (pending인 경우)
```
