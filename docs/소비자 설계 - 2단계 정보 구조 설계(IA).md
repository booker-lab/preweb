## 1. 사이트맵 (Sitemap)

### Main Navigation (하단 탭 5개)

```
├── 홈 (Home)
│     주요 큐레이션, 실시간 인기 상품(수동/자동 혼합), 이벤트 배너
│
├── 카테고리 (Category)
│     품목별 체계적 분류 (대/중/소분류 계층 구조)
│
├── 검색 (Search)
│     상품 키워드 검색 (상품명, 꽃말 등)
│
├── 장바구니 (Cart)
│     구매 예정 상품 목록 및 수량 조절
│
└── 마이페이지 (My Page)
      내 정보, 주문 현황, 고객센터
      PWA 홈 화면 추가 버튼 (A2HS)
```

### Sub Screens (상세 및 기능 화면)

```
├── 로그인 (Login)                            ← 독립 탭 없음, 아래 두 시점에서만 진입
│     진입 시점 A: 미로그인 상태에서 결제하기 클릭
│     진입 시점 B: 미로그인 상태에서 마이페이지 탭 클릭
│     옵션: 카카오 / 네이버 소셜 로그인
│           이메일 로그인 / 회원가입
│           (비회원 구매 미지원)
│
├── 상품 상세 (Product Detail)
│     상품 설명, 실시간 시세 정보, 소싱 프로세스 안내
│     [일반 판매] 구매하기 / 장바구니 버튼
│     [공동구매] 실시간 참여 인원 현황 + 참여하기 버튼
│
├── 결제 (Checkout)
│     배송지 입력, 배송 수단 선택, 배송 희망일 선택, 배송비 확인, 결제 수단 선택 (Portone)
│
├── 주문 완료 (Order Success)
│     결제 성공 확인 및 주문 현황 이동 버튼
│     [공동구매] "모집이 완료되면 주문이 확정됩니다" 안내
│
└── 주문 현황 보드 (Order Status)
      판매자·드라이버 앱과 연동된 실시간 단계별 진행 상황 피드
      [공동구매] 모집 중 상태: 실시간 참여 인원 카운트 + 마감 타이머
```

## 2. 사용자 이동 경로 (User Flow)

### A. 구매 경로 (일반 판매)

```
홈 / 카테고리
    ↓
상품 상세 (saleType: normal)
    ↓
장바구니 담기
    ↓
[미로그인 상태] → 로그인 화면 (소셜 / 이메일)
[로그인 상태]   → 바로 결제 페이지 진입
    ↓
결제 페이지
  - 배송지 입력 (저장된 배송지 선택 또는 신규 입력)
  - 배송 수단 선택 (isMetropolitan 자동 필터링)
  - 배송 희망일 선택 (Date Picker)
  - 배송비 자동 계산 표시
  - 결제 수단 선택
    ↓
결제 완료 → 상태: ACCEPTED (결제 완료)
    ↓
주문 완료 화면 → 실시간 주문 현황 보드
```

### B. 구매 경로 (공동구매)

```
홈 / 카테고리
    ↓
상품 상세 (saleType: group)
  - 현재 참여 인원 / 최소 인원 실시간 표시
  - 마감까지 남은 시간 표시
  ※ 장바구니 없음 → 참여하기 클릭 시 바로 결제 페이지 진입
    ↓
[미로그인 상태] → 로그인 화면 (소셜 / 이메일)
[로그인 상태]   → 바로 결제 페이지 진입
    ↓
결제 페이지
  - 배송지 입력
  - 배송 수단 표시 (소비자 선택 불가, 판매자 지정 / isMetropolitan 기반 자동 표시)
  - 배송 예정일 표시 (판매자가 상품 등록 시 지정한 날짜 → 소비자 변경 불가)
  - 배송비 표시 (할인 배송비, 이벤트 시 무료)
  - 결제 수단 선택
  - 안내 문구: "목표 인원 미달 시 자동 환불됩니다"
    ↓
결제 완료 → 상태: RECRUITING (모집 중)
    ↓
주문 완료 화면
  - "현재 N명 참여 중, 최소 M명이 모이면 확정됩니다"
    ↓
실시간 주문 현황 보드 (모집 중 상태 유지)
    ↓
[최소 인원 달성] → CONFIRMED (주문 확정) → PREPARING → DELIVERING → DELIVERED
[마감까지 미달]  → CANCELLED (자동 환불, 알림톡 발송)
```


### C. 미로그인 상태에서 행동 시도 시

```
결제하기 클릭 또는 마이페이지 탭 클릭
    ↓
로그인 화면으로 이동
  - 카카오 / 네이버 소셜 로그인
  - 이메일 로그인 / 회원가입
    ↓
로그인 완료 → 원래 진입하려던 화면으로 복귀
```

## 3. 상세 화면별 주요 구성 요소

### 1. 홈 (Home)

```
상단: 알림 아이콘 (위치 설정 UI 없음 - MVP 제외)
중단: 이벤트 배너, 오늘의 시세 급변동 상품
하단: 오늘의 베스트 상품 리스트
```

> 지역 기반 필터링은 다중 판매자 전환(Phase 2) 이후 추가 예정

---

### 2. 카테고리 (Category)

```
헤더
  - 좌: "카테고리" 제목
  - 우: 검색 아이콘 (클릭 시 검색 페이지로 이동)

카테고리 탭 (단일 선택)
  전체 | 절화 | 난 | 관엽

색상 필터 (카테고리 탭 하단, 멀티 선택 가능)
  [기본: 접힌 상태 - "색상 ▾" 버튼으로 펼치기]
  펼친 상태:
    레드  핑크  화이트  옐로우  오렌지
    퍼플  블루  그린    무늬    브라운
    베이지 블랙 그레이
  - 선택된 색상: 해당 색상 원형 + 초록 테두리 활성화
  - 멀티 선택 가능 (예: 레드 + 핑크 동시)
  - 카테고리 탭 AND 조건 조합 (예: 절화 + 레드)
  - 초기화 버튼으로 필터 전체 해제

정렬 바
  - 좌: "총 OOO개"
  - 우: "최신순 ▾" 드롭다운 (최신순 / 인기순 / 낮은가격순 / 높은가격순)

상품 그리드
  - 2열 카드 레이아웃
  - 홈 화면과 동일한 카드 디자인
  - 공동구매 상품: 진행 바 + 참여인원 + 마감일 표시
```

**판매자 앱 연동**: 상품 등록 시 카테고리(절화/난/관엽) + 색상(멀티) 필드 입력 → Firestore products.colors 저장 → 소비자 앱 색상 필터에 반영

### 7. 결제 (Checkout)

**[공통]**
```
주문자 정보
  - 이름, 연락처 (자동 입력 - 회원)

배송지
  - 저장된 배송지 선택 (회원) / 신규 입력
  - 상세 주소 입력
  - 주소 입력 즉시 isMetropolitan 판별 (서울/경기 여부 자동 확인)

배송 수단 선택 (배송지 입력 후 활성화)
  - isMetropolitan = true (서울/경기):
      [꽃차 직배송] [거점 픽업] [택배] 중 선택
  - isMetropolitan = false (기타 지역):
      [택배]만 표시 (직배송·거점 픽업 자동 비활성)
  - 기상 조건 제한 활성 시: [택배] 옵션 자동 비활성화
      안내 문구: "현재 기온 조건으로 인해 택배 배송이 일시 중단되었습니다"

배송비 (배송 수단 및 품목 사이즈에 따라 자동 계산)
  - 무료 배송 기준 충족 시 "배송비 무료" 표시
  - 상품 금액 + 배송비 합산 표시

결제 수단
  - 카카오페이
  - 네이버페이
  - 신용/체크카드
  (무통장 입금 미지원)

최종 결제 금액 확인 (상품 금액 + 배송비) + 결제하기 버튼
```

**[일반 판매 saleType: normal]**
```
배송 희망일 선택 (Date Picker)
  - 활성 날짜: 배송 가능한 날짜만 선택 가능
  - 비활성 날짜: Daily Cap 초과 시 Sold Out 표시
  - 품목별 별도 마감 로직 적용 가능
```

**[공동구매 saleType: group]**
```
배송 수단 표시 (소비자 선택 불가, 자동 결정)
  - 서울/경기: 꽃차 직배송 또는 택배 (판매자가 상품 등록 시 지정)
  - 기타 지역: 택배

배송비: 일반 배송비 대비 할인 적용 (판매자가 상품 등록 시 설정)
  - 기본 원칙: 일반 판매보다 낮은 배송비로 참여 유도
  - 이벤트/프로모션 시 무료 배송 설정 가능 (판매 촉진 도구로 활용)

배송 예정일 표시 (판매자가 상품 등록 시 지정한 날짜 → 소비자 변경 불가)
안내 문구: "목표 인원 미달 시 자동 환불됩니다"
```

### 9. 주문 현황 보드 (Order Status)

**[일반 판매 - 꽃차 직배송 / 택배]**
```
상단 고정: "선택하신 [M월 D일]에 배송될 예정입니다"

상태 바:
  결제 완료 → 상품 준비 중 → 배송 중 → 배송 완료
```

**[일반 판매 - 거점 픽업 (Model B)]**
```
상단 고정: "선택하신 [M월 D일]부터 수령 가능합니다"

픽업 코드 (HUB_ARRIVED 단계부터 표시):
  ┌──────────────────────────────┐
  │  픽업 코드: 123456            │
  │  수령 장소: OO 꽃집 (주소)    │
  │  코드를 제시하고 수령하세요    │
  └──────────────────────────────┘

상태 바:
  결제 완료 → 상품 준비 중 → 배송 중 → 거점 도착 → 픽업 완료

상태 코드 매핑:
  결제 완료  = ACCEPTED
  상품 준비 중 = PREPARING
  배송 중    = DELIVERING  (드라이버가 거점으로 이동 중)
  거점 도착  = HUB_ARRIVED (거점 도착, 픽업 코드 활성화)
  픽업 완료  = PICKED_UP   (소비자 수령 완료 → REVIEWED 전환 가능)
```

**[공동구매 - 모집 중]**
```
상단 고정: "모집 마감: M월 D일 / 배송 예정: M월 D일"

모집 현황:
  ██████████░░  18/30명 참여 중
  "2명만 더 모이면 주문이 확정됩니다!"
  마감 카운트다운 타이머

상태 바:
  [모집 중] → 주문 확정 → 상품 준비 중 → 배송 중 → 배송 완료
```

**[공동구매 - 확정 이후]**
```
상단 고정: "선택하신 [M월 D일]에 배송될 예정입니다"

상태 바:
  모집 중 → [주문 확정] → 상품 준비 중 → 배송 중 → 배송 완료
```

Firestore 실시간 리스너로 참여 인원 및 상태 자동 갱신

### 10. 마이페이지 (My Page)

```
[로그인 상태]
  - 프로필 정보
  - 나의 주문 내역 (일반 판매 주문)
      DELIVERED 또는 PICKED_UP 상태 주문 → "구매 확정" 버튼 + "후기 작성" 버튼 노출
        구매 확정 클릭 → 상태 REVIEWED로 전환
        후기 작성 클릭 → 후기 작성 화면 진입 → 작성 완료 시 상태 REVIEWED로 전환
        (둘 중 하나만 해도 REVIEWED 전환)
  - 나의 공동구매 참여 내역 (모집 중 / 확정 / 취소 / 완료 상태별 확인)
      DELIVERED 상태 공동구매 → 동일하게 "구매 확정" + "후기 작성" 버튼 노출
      (공동구매는 직배송·택배 전용이므로 PICKED_UP 미해당)
  - 배송지 관리
  - 고객센터
  - PWA 홈 화면 추가 버튼 (A2HS)
      미설치 상태 → "홈 화면에 추가" 버튼 노출
```

### 공동구매 알림 시나리오

| 시점 | 알림 내용 |
|------|----------|
| 참여 완료 | "OO 상품 모집에 참여하셨습니다. 현재 18/30명" |
| 마감 임박 (2시간 전) | "마감 2시간 전, N명이 더 필요해요. 주변에 공유해보세요!" |
| 목표 달성 | "목표 인원이 모였습니다! 주문이 확정되었습니다." |
| 목표 미달 취소 | "[목표 수량 미달성으로 취소] 결제 금액은 3~5일 내 환불됩니다." |

### 취소(CANCELLED) 소비자 알림 워딩 원칙

취소 사유를 소비자가 명확히 이해할 수 있는 워딩으로 표시합니다.

```
취소 사유 예시 워딩:
  - "[목표 수량 미달성으로 취소]" - 최소 주문 수량 미충족 시
  - "[산지 재고 소진으로 취소]"   - 해당 품목 물량 확보 실패 시
  - "[배송 불가 지역으로 취소]"   - 배송 범위 외 주소 확인 시
  - "[결제 오류로 취소]"          - 결제 처리 실패 시

전달 방식:
  → 카카오 알림톡으로 취소 사유 즉시 발송
  → 주문 현황 보드에 취소 사유 텍스트 함께 노출
  → 환불 처리 일정 함께 안내
```

## 5. 물류 캐파 관리 로직

### Daily Cap 설정 위치

```
설정 주체: 판매자 (디어 오키드)
설정 위치: 판매자 앱 → 배송 관리 → 일별 캐파 설정 메뉴
```

### 동작 방식

```
1. 판매자 앱에서 날짜별 최대 주문 수량(Daily Cap) 설정
   예) 2026-03-25: 최대 30건

2. 소비자가 결제 화면의 Date Picker에서 날짜 선택 시
   → Firestore에서 해당 날짜 잔여 슬롯 실시간 조회

3. 잔여 슬롯 > 0  → 날짜 활성화 (선택 가능)
   잔여 슬롯 = 0  → 날짜 비활성화 (Sold Out 표시)

4. 결제 최종 확인 시점에 슬롯 재검증
   → 동시 주문으로 슬롯 소진 시 결제 차단 + 다른 날짜 선택 유도
```

### 거점 픽업 Daily Cap 적용

```
MVP 단계 인력 부족 상황을 고려하여
거점 픽업(Model B) 주문도 일반 배송과 동일한 Daily Cap 적용

  → 판매자 앱에서 날짜별 전체 수용량(직배송 + 거점 픽업 합산) 설정
  → Daily Cap 초과 시 거점 픽업 날짜도 동일하게 비활성화
```

### 가변적 마감 로직

```
특정 품목(경매 물량이 적은 꽃 등)은
품목별 별도 마감 로직 적용 가능하도록 설계
  → 품목 단위 재고 제한 설정 (판매자 앱에서 관리)
```
